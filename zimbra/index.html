Zimbra

Zimbra Collaboration Suite is a free open source full-featured messaging and collaboration solution that includes email, contacts , calendaring and tasks. Zimbra Collaboration Suite is built with well known open source technologies and standard based protocols. Zimbra open source application consists of client interfaces and server components.

Zimbra architecture contain the following :-

a) Open source integration : Linux, Postfix, MariaDB, OpenLdap, Jetty, Nginx, openDKIM

b) Industry standard protocols : SMTP,LMTP,IMAP,IMAPS,POP,POPS

c) Modern Technology design : Java, HTML, XML, Javascript

d) Browser Based client interfaces : Zimbra web client gives users access to all zimbra features

Zimbra Components

Zimbra uses Some third party software which is bundled with zimbra software and installed as part of installation process. Some of the Components are as below.

Jetty : The web application server that Zimbra software runs in.

MariaDB : Database software

Lucene : An open source full-featured text and search engine

Postfix : An open source mail transfer agent (MTA) that routes mail messages to the appropriate Zimbra server

OpenLDAP : An open source implementation of the Lightweight Directory Access Protocol (LDAP) that stores Zimbra system configuration, the Zimbra Global Address List, and provides user authentication. Zimbra can also work with GAL and authentication services provided by external LDAP directories such as Active Directory

ClamAV : An anti-virus scanner that protects against malicious files

SpamAssassin : A mail filter that attempts to identify spam

Amavis : Interfaces between the MTA and one or more content checkers

Zimbra core : Includes the libraries, utilities, monitoring tools, and basic configuration files. zmconfigd is part of zimbra-core and is automatically enabled and runs on all systems.

Zimbra MTA : Postfix is the open source mail transfer agent (MTA) that receives email via SMTP and routes each message to the appropriate . Zimbra mailbox server use Local Mail Transfer Protocol (LMTP). The Zimbra MTA also includes the anti-virus and anti-spam components.

Zimbra Proxy : Zimbra Proxy is a high-performance reverse proxy service for passing IMAP[S]/POP[S]/HTTP[S] client requests to other internal ZCS services. This package is normally installed on the MTA server(s) or on its own independent server(s). When the zimbra-proxy package is installed, the proxy feature is enabled by default. Installing the Zimbra Proxy is highly recommended, and required if using a separate web application server.

Zimbra SNMP : The Zimbra SNMP package is optional. If you choose to install zimbra-SNMP for monitoring, this package should be installed on every Zimbra server.

Zimbra Logger : The Zimbra Logger package is optional and is installed on one mailbox server. The Zimbra Logger installs tools for syslog aggregation and reporting. If you do not install Logger, the server statistics section of the administration console will not display. The Logger package must be installed at the same time as the mailbox server.

Zimbra Spell : The Zimbra Spell package is optional. A spell is the open source spell checker used on the Zimbra Web Client. When Zimbra-Spell is installed, the Zimbra-Apache package is also installed.

spacewalk client

Spacewalk is an open source (GPLv2) Linux systems management solution. It is the upstream community project for Red Hat Satellite 5. Its capabilities include:

  1. Inventory your systems (hardware and software information)
  2. Install and update software on your systems
  3. Collect and distribute your custom software packages into manageable groups
  4. Provision (kickstart) your systems
  5. Manage and deploy configuration files to your systems
  6. Provision virtual guests
  7. Start/stop/configure virtual guests
  8. Distribute content across multiple geographical sites in an efficient manner

Procedure to configure spacewalk client to join to its server

1. Make a script file script.sh in root. copy following into the scriptfile
#Install Repositories
yum install -y yum-plugin-tmprepo
yum install -y spacewalk-client-repo --tmprepo=https://copr-be.cloud.fedoraproject.org/results/%40spacewalkproject/spacewalk-2.9-client/epel-7-x86_64/repodata/repomd.xml --nogpg
rpm -Uvh http://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

#Install Spacewalk Client Packages
yum -y install rhn-client-tools rhn-check rhn-setup rhnsd m2crypto yum-rhn-plugin rhncfg-actions osad

#Install Spacewalk CA certificate
rpm -Uvh http://space.f1soft.com/pub/rhn-org-trusted-ssl-cert-1.0-6.noarch.rpm

#Register to spacewalk
rhnreg_ks --serverUrl=https://space.f1soft.com/XMLRPC --sslCACert=/usr/share/rhn/RHN-ORG-TRUSTED-SSL-CERT --activationkey=1-space-centos7-base 

#Enable and Start osad and rhnsd service
systemctl enable osad rhnsd && systemctl start osad rhnsd

#Enable action to deploy updates and run scripts from satellite server
rhn-actions-control --enable-all

#Disable default CentOS-Base and Updates and epel repos
#This is done for the clients to prevent updates from internet
#And install/get updates from spacewalk repositories
yum-config-manager --disable updates extras base epel duosecurity
2. Make the script executable
chmod +x space.sh
3. Run the script from root
./space.sh
4. Login to space.f1soft.com in browser navigate to system and verify wheather the client is listed

5. select the configured client server and add required repo from subscribed channel area.

Ansible

Ansible is an open source automation software written in Python. It is an agentless automation tool that by default manages machines over the SSH protocol. Modules can be written in any language and communicate in standard JSON. Configuration files are mainly written in the YAML data serialization format. Ansible can interact with clients through either command line tools or through its configuration scripts called Playbooks.

In this guide, youâ€™ll install Ansible on a CentOS 7 server and learn some basics of how to use. We have following server/client detail.

Ansible Server : Centos 7, 192.168.91.50
Client  : Centos 7, 192.167.91.51

We need to install EPEL repository to get ansible. Run the following command on ansible server.

yum install epel-repo -y

Install ansible using,

yum install ansible -y

Check ansible version, the version should be displayed.

ansible --version

Although we can connect to remote hosts using a password through Ansible it is recommended to set up key-based authentication for easy and secure logins. Generate an SSH key pair on your system with empty passphrase, by running the command.

ssh-keygen

Copy Public Key into Target Server.

ssh-copy-id root@192.168.91.51

By default, Ansible reads the host file from the location /etc/ansible/hosts. Add following in the file,

[servers]
server1 ansible_host=192.168.91.51 ansible_user=root

Test the connection,

ansible all -m ping

It should show success to server1.

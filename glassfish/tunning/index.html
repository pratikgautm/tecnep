Glassfish Tunning

While Installing Glassfish server may seems easy, sometimes you may face lots of issue due to glassfish server not being finely tunned. By default configuration, glassfish server comes with the minimal optipons configured. Here let's discuss about tunning the glassfish server for your environment. Let's first disscuss about the JVM options. bin/asadmin –port list-jvm-options

a. List all the JVM Options configured for domain

bin/asadmin --port <admin-port> list-jvm-options

b. From above command you will get a list of JVM options. Example being as follow.

c. Now from above results, let us take multiple JVM options which are of utmost importance for your performance of the server.

  1. Heap Memory.

Heap memory is the run time data area from which the memory for all java class instances and arrays is allocated. From this we can see the importance of the heap memory. In glassfish we can configure two important jvm options for heap memory. One is initial heap size which is refered as “-Xmx1500m” and another is maximum heap size which is refered as “-Xms1500m”.

d. Encrypted Password for DataSource

  1. View List for encrypted password

     bin/asadmin --port <admin-port> list-password-aliases

This command will prompt you to enter the admin username & password, after successful authentication, you can see the list of encrypted password which were previously created.

  1. Create encrypted password

     bin/asadmin --port <admin-port> create-password-alias

This command will prompt you to enter the admin username & password, after successful authentication, it will ask you for “Name” of Alias Password and then it will ask you to enter the desired “Password” two times. If the entered password matches two times then, it will generate a message with “Password-Alias created successfully”. Now, you can use the recently created password-alias “Name” for Database Connection at JDBC Connection Pool. You need to replace the password at password field with this “${ALIAS=alias_name}".

  1. Delete encrypted password

     bin/asadmin --port <admin-port> delete-password-alias

This command will prompt you to enter the admin username & password, after successful authentication, it will ask you for “Name” of Alias Password that you wish to delete.

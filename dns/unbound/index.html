Unbound

Unbound is a recursive, and caching DNS server. Caching name servers store DNS query results for a period of time determined in the configuration (time-to-live) of the domain name record in question.

Here in this article, we are going to use ‘unbound‘ caching software to install and configure a DNS Server in CentOS 7 systems.

Install it

yum install unbound -y

Make a copy of the unbound configuration file before making any changes to original file.

mv /etc/unbound/unbound.conf /etc/unbound/unbound.conf.original

Add the following parameters in config file /etc/unbound/unbound.conf

server:
  access-control: 192.168.222.0/24 allow_snoop   # to allow query
  cache-max-ttl: 14400                        # maximum time to live cache
  cache-min-ttl: 900                          # minimum time to live cache
  hide-identity: yes
  num-threads: 2
  do-ip6: no
  hide-version: yes
  interface: 192.168.222.10               # interface ip
  interface: 127.0.0.1
  minimal-responses: yes
  log-servfail: yes
  jostle-timeout: 4000
  msg-cache-slabs: 4
  module-config: "validator iterator"
  auto-trust-anchor-file: "root.key"
  directory: "/etc/unbound"
  delay-close: 10000
  infra-host-ttl: 60
  infra-cache-min-rtt:5000
  rrset-roundrobin: yes
  log-queries: yes
  logfile: "/log/unbound.log"             # log file
  chroot: "/etc/unbound"
  verbosity: 1
  root-hints: root.hints
remote-control:
  control-enable: yes

Start and enable the service

systemctl start unbound && systemctl enable unbound

Verify using,

unbound-control status

See the statics,

unbound-control stats

Storage

Storage Types
1. DAS

Direct-attached storage (DAS) is digital storage directly attached to the computer accessing it. Examples of DAS are hard drives, solid-state drives, optical disc drives. The main protocols used for DAS connections are PATA, SATA, IDE, NVMe, SCSI, SAS, USB.

  1. PATA : Data is transferred in parallel. They are older technology and slower than SATA. PATA interface doesn’t support hot plugging.

  2. IDE : IDE is also called Parallel ATA or PATA.

  3. SATA : Serial Advanced Technology Attachment (SATA) is the newer standard and SATA drives are faster than PATA drives. SATA interface supports hot plugging.

  4. SAS : Serial Attached SCSI is a point-to-point serial protocol that moves data to and from computer-storage devices such as hard drives and tape drives. It is morden SCSI.

  5. NVMe : It is open logical device interface specification for accessing non-volatile storage media attached via PCI Express (PCIe) bus.

2. NAS
  • Network-attached storage is a file-level computer data storage server connected to a computer network providing data access to a group of clients.
  • It uses TCP/IP networks, commonly ethernet.
  • They typically provide access to files using network file sharing protocols such as NFS, SMB.
  • Due to the use of ethernet, has lower throughput and higher latency.
  • Some large NAS manufacturers like Synology, QNAP.
3. SAN
  • A Storage Area Network (SAN) is a specialized, high-speed network that provides block-level network access to storage.
  • They uses fiber to transfer data.
  • SANs are commonly based on Fibre Channel (FC) technology that utilizes the Fibre Channel Protocol (FCP).
  • Also can be implemented using Fibre Channel over Ethernet (FCoE), which are cheaper.
  • Due to the use of fiber, has higher throughput and lower latency.
  • Some SAN vendors are Dell EMC, HPE, NetApp, IBM
File system

A file system controls how data is stored and retrieved. Without a file system, there is no way to tell where one piece of data stops and the next begins. There are many different kinds of file systems. Each one has different structure, properties of speed, security, size. Some of them are explained below:-

a) ext4

  • used by linux system.
  • designed to be backwards compatible with ext3, ext2.
  • uses hashed B-tree data structure to store and retrieve data.
  • maximum Volume size = 1 Ebyte, max file size = 16 Tbytes.

b) xfs

  • default file system in centos 7 and RHEL 7.
  • can be enlarged, but not shrunk, on the fly.
  • uses B+ tree data structure to store and retrieve data.
  • maximum Volume size = 8 Ebytes, max file size = 8 Ebytes.

c) NTFS

  • a high-performance file system proprietary to Microsoft.
  • linux have open-source NTFS driver, with both read and write functionality.
  • macOS comes with read-only support for NTFS.
  • NTFS replaced High-Performance File System (HPFS) and the Windows 95 file allocation table (FAT).
